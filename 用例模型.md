# 用例模型 —— 生物实验助手

## 1. 参与者（Actor）

| Actor | 描述 |
|-------|------|
| 实验负责人 | 创建和管理实验项目的用户，具有最高项目管理权限 |
| 实验操作员 | 执行实验、按计划采集并记录数据的用户 |
| 数据分析员 | 对实验数据进行统计分析和报告生成的用户（可与负责人为同一人） |
| 协作者 | 被邀请参与项目的其他用户，可查看和编辑实验数据 |
| 系统 | 定时触发提醒、调用Agent、备份草稿的后台服务 |

## 3. 用例详细描述

### UC-01：管理实验项目

| 属性 | 说明 |
|------|------|
| 用例编号 | UC-01 |
| 用例名称 | 管理实验项目 |
| 参与者 | 实验负责人 |
| 前置条件 | 用户已登录系统 |
| 后置条件 | 项目创建/修改/删除成功，数据持久化 |
| 基本流程 | 1. 用户进入项目列表页<br>2. 用户点击"新建项目"<br>3. 填写项目名称、描述、实验目的等基本信息<br>4. 系统验证必填字段，保存项目<br>5. 返回项目列表，显示新项目 |
| 扩展流程 | 3a. 用户可选择从模板创建项目（后续迭代）<br>4a. 必填字段为空时提示用户补充<br>4b. 用户可取消创建，返回列表 |
| 特殊需求 | 支持项目搜索、按时间/状态排序 |

### UC-02：定义数据条目

| 属性 | 说明 |
|------|------|
| 用例编号 | UC-02 |
| 用例名称 | 定义数据条目 |
| 参与者 | 实验负责人 |
| 前置条件 | 已创建实验项目 |
| 后置条件 | 数据条目定义保存，属于该项目的记录模板生效 |
| 基本流程 | 1. 用户进入某个项目详情页<br>2. 点击"数据条目管理"<br>3. 添加数据条目：名称、数据类型（数值/文本/图片）、单位、是否必填<br>4. 可拖拽调整条目顺序<br>5. 保存条目定义 |
| 扩展流程 | 3a. 支持批量导入条目（CSV）<br>5a. 保存时系统校验条目名称唯一性 |
| 特殊需求 | 条目支持分组（如：环境参数组、样品参数组） |

### UC-03：记录实验数据

| 属性 | 说明 |
|------|------|
| 用例编号 | UC-03 |
| 用例名称 | 记录实验数据 |
| 参与者 | 实验操作员 |
| 前置条件 | 项目中已定义数据条目，用户有权访问该项目 |
| 后置条件 | 数据记录保存成功 |
| 基本流程 | 1. 用户进入项目，点击"添加记录"<br>2. 系统展示已定义的数据条目表单<br>3. 用户逐项填写数据，可上传附图（照片/截图）<br>4. 系统自动记录填写时间戳<br>5. 用户保存记录<br>6. 系统持久化存储，返回记录列表 |
| 扩展流程 | 3a. 数值型条目校验格式（数字范围）<br>3b. 图片上传支持拍照/相册选择<br>5a. 保存失败时提示错误原因<br>5b. 编辑器崩溃，下次打开恢复草稿 |
| 特殊需求 | 支持离线草稿自动保存（LocalStorage/IndexedDB） |

### UC-04：采集数据提醒

| 属性 | 说明 |
|------|------|
| 用例编号 | UC-04 |
| 用例名称 | 采集数据提醒 |
| 参与者 | 实验操作员、系统 |
| 前置条件 | 项目中已设定采集时间计划 |
| 后置条件 | 用户在预设时间收到提醒 |
| 基本流程 | 1. 用户在项目设置中规划采集时间（支持单次/周期）<br>2. 系统保存提醒计划<br>3. 到达预设时间前（如提前5分钟），系统推送通知<br>4. 用户收到提醒，进入项目记录数据 |
| 扩展流程 | 2a. 支持设置提前提醒时间（5分钟/15分钟/30分钟）<br>3a. 用户可标记"已完成"或"稍后提醒"<br>3b. 若超时未处理，系统周期性重复提醒（可配置间隔） |
| 特殊需求 | 提醒需支持浏览器通知（Notification API）、邮件通知 |

### UC-05：Agent数据分析

| 属性 | 说明 |
|------|------|
| 用例编号 | UC-05 |
| 用例名称 | Agent数据分析 |
| 参与者 | 数据分析员 |
| 前置条件 | 项目中已有实验数据记录 |
| 后置条件 | 获得分析结果，可导出报告 |
| 基本流程 | 1. 用户进入项目数据分析页<br>2. 选择待分析的数据条目和时间范围<br>3. 可选：从预设列表选择数学模型（线性回归/多项式拟合/指数模型/对数模型/sigmoid等）<br>4. 点击"开始分析"<br>5. 系统调用Agent API，传入数据和分析要求<br>6. Agent返回：模型优化参数、相关性分析（R²等）、趋势解读<br>7. 系统展示分析结果（图表+文字） |
| 扩展流程 | 3a. 用户不选择模型，Agent自动推荐最优拟合<br>5a. Agent调用超时/失败时提示用户重试<br>6a. 结果不满意可调整参数重新分析 |
| 特殊需求 | 分析结果可视化（图表展示拟合曲线与原始数据对比） |

### UC-06：生成实验报告

| 属性 | 说明 |
|------|------|
| 用例编号 | UC-06 |
| 用例名称 | 生成实验报告 |
| 参与者 | 数据分析员 |
| 前置条件 | 已完成数据分析（UC-05）或项目中已有足够数据 |
| 后置条件 | 生成可导出的实验报告 |
| 基本流程 | 1. 用户点击"生成报告"<br>2. 选择报告模板（完整报告/摘要报告）<br>3. 系统汇总：项目信息、数据记录、分析结果、图表<br>4. Agent生成摘要与结论<br>5. 用户预览报告<br>6. 导出为PDF/Word |
| 扩展流程 | 3a. 用户可手动编辑报告内容<br>6a. 支持直接打印 |
| 特殊需求 | 报告包含实验信息、数据表格、分析图表、结论建议 |

### UC-07：查询公开数据

| 属性 | 说明 |
|------|------|
| 用例编号 | UC-07 |
| 用例名称 | 查询公开数据 |
| 参与者 | 所有用户 |
| 前置条件 | 网络连接正常 |
| 后置条件 | 获取公开实验数据/流程信息 |
| 基本流程 | 1. 用户进入"公开数据"页面<br>2. 输入关键词搜索（实验名称、方法、试剂等）<br>3. 系统调用公开数据库/搜索引擎API<br>4. 展示匹配的公开实验流程与数据<br>5. 用户可查看详情，或导入到自己的项目中作为参考 |
| 扩展流程 | 3a. 无匹配结果时给出提示<br>5a. 导入时标记数据来源 |
| 特殊需求 | 支持按学科领域、发布时间等筛选 |

### UC-08：上传与分享数据

| 属性 | 说明 |
|------|------|
| 用例编号 | UC-08 |
| 用例名称 | 上传与分享数据 |
| 参与者 | 实验负责人 |
| 前置条件 | 项目中已有完整数据 |
| 后置条件 | 数据公开发布或分享给指定用户 |
| 基本流程 | 1. 用户进入项目设置<br>2. 点击"分享/发布"<br>3. 选择分享方式：公开发布 / 链接分享 / 邀请协作者<br>4. 确认隐私设置（脱敏处理）<br>5. 系统生成分享链接或发布数据 |
| 扩展流程 | 3a. 邀请协作者需输入对方账号/邮箱<br>4a. 提示用户检查是否有敏感数据 |
| 特殊需求 | 分享链接支持设置有效期和访问密码 |
