# 技术选型与学习计划 —— 生物实验助手

## 1. 技术选型总览

| 层级 | 技术 | 版本 | 选型理由 |
|------|------|------|----------|
| 前端框架 | React | 18.x | 生态成熟、组件化开发效率高、社区资源丰富、适合SPA |
| 构建工具 | Vite | 5.x | 极速冷启动、HMR热更新、原生ESM支持 |
| UI组件库 | Ant Design | 5.x | 企业级组件丰富、表格/表单/图表开箱即用、适合数据密集型应用 |
| 状态管理 | Zustand | 4.x | 轻量级、API简洁、TypeScript友好 |
| 路由 | React Router | 6.x | React生态标准路由方案 |
| 图表 | ECharts | 5.x | 强大的数据可视化能力、支持科学图表 |
| HTTP客户端 | Axios | 1.x | 拦截器、请求取消、上传进度等 |
| 后端框架 | Node.js + Express | LTS | 前后端统一语言、生态丰富、适合中小规模API |
| 数据库 | PostgreSQL + Prisma ORM | 16 | 可靠的关系型数据库、Prisma提供类型安全的ORM |
| 定时任务 | node-cron | - | 轻量级定时任务调度，适合提醒功能 |
| Agent对接 | OpenAI API / 兼容接口 | - | 调用大语言模型进行数据分析 |
| 文件存储 | 本地FS + 云存储预留 | - | 图片存储，预留OSS接口 |
| 实时通知 | WebSocket (Socket.io) | 4.x | 实时推送提醒通知 |
| 认证 | JWT + bcrypt | - | 无状态认证，适合前后端分离 |
| 草稿恢复 | IndexedDB (Dexie.js) | 3.x | 浏览器端本地持久化，崩溃恢复 |
| 测试 | Vitest + Playwright | - | 单元测试 + E2E测试 |

## 2. 备选方案对比

## 4. 学习计划

### 4.1 第一阶段：基础搭建（第1-2天）

| 任务 | 内容 | 学习资源 |
|------|------|----------|
| React + Vite项目脚手架 | 使用Vite创建React+TS项目，配置ESLint/Prettier | Vite官方文档 |
| Ant Design入门 | 学习Layout/Menu/Table/Form/Button等核心组件 | Ant Design官方文档 |
| React Router配置 | 配置基础路由：首页/项目列表/项目详情/数据分析 | React Router官方文档 |
| Zustand状态管理 | 学习基本store创建和使用 | Zustand GitHub |

### 4.2 第二阶段：后端搭建（第3-4天）

| 任务 | 内容 | 学习资源 |
|------|------|----------|
| Express基础 | RESTful API设计、中间件机制、错误处理 | Express官方文档 |
| Prisma + PostgreSQL | Schema设计、CRUD操作、Migration | Prisma官方文档 |
| JWT认证 | 注册/登录/Token刷新流程 | JWT.io |

### 4.3 第三阶段：核心功能开发（第5-8天）

| 任务 | 内容 | 学习资源 |
|------|------|----------|
| 提醒系统 | node-cron定时任务 + Socket.io实时推送 + 浏览器Notification API | 各官方文档 |
| 文件上传 | Multer文件上传、前端Ant Design Upload组件 | Multer/Antd Upload文档 |
| Agent对接 | OpenAI API调用、流式输出处理、超时重试 | OpenAI API文档 |
| ECharts集成 | 折线图/散点图/拟合曲线等科学图表 | ECharts官方文档 |

### 4.4 第四阶段：质量保障（第9-10天）

| 任务 | 内容 | 学习资源 |
|------|------|----------|
| IndexedDB草稿 | Dexie.js实现编辑器草稿自动保存与恢复 | Dexie.js文档 |

| 性能优化 | React.memo/虚拟列表/图片懒加载/API响应缓存 | React性能优化指南 |
| 基础测试 | Vitest单元测试（核心工具函数）+ Playwright E2E（关键流程） | Vitest/Playwright文档 |

## 5. 技术风险与应对

| 风险 | 概率 | 影响 | 应对措施 |
|------|------|------|----------|
| Ant Design表单复杂度超出预期 | 中 | 中 | 提前学习动态表单(Form.List) |
| Agent API调用延迟高 | 高 | 高 | 前端loading状态+超时提示+后台异步分析 |
| IndexedDB兼容性问题 | 低 | 中 | 检测浏览器支持，降级为LocalStorage |
| PostgreSQL新手学习成本 | 中 | 中 | Prisma ORM降低SQL门槛，优先CRUD |
| Socket.io在移动端稳定性 | 中 | 中 | 增加HTTP轮询降级方案 |
