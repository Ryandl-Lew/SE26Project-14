## 🌟 核心功能特性一览

### 1. 📁 文件管理与回收站全闭环 (File Management & Trash Bin)
重构了文件上传、预览、下载及生命周期管理：
* **彻底的软删除闭环**：
  * **后端防误删**：抛弃了传统的“一删到底”模式，引入 JPA `@SQLRestriction` 实现逻辑删除。删除动作仅标记 `deleted=true`，严格保留物理文件留作容错缓冲，配合定时任务进行物理回收。
  * **JPA 缓存防御**：在文件下载/预览 API 中加入了对 `deleted` 状态的强校验，防止 JPA 一级缓存导致“僵尸文件”复活。
* **自适应 UI 与极客交互**：
  * **Fluid Flexbox 布局**：彻底解决文件列表在窄屏卡片容器中的溢出挤压问题。
  * **动态紧凑模式 (`compact`)**：在侧边栏自动隐藏次要信息，结合 `Ellipsis` 文本截断与 **Hover Tooltip (悬浮气泡)**，在极小空间内优雅展示长文件名、大小与上传时间。
  * **沙箱预览与格式校验**：支持 `Content-Disposition: inline` 的安全图片/PDF 行内弹窗预览；采用 UUID 随机混淆重命名，彻底规避磁盘重名与 `../` 路径穿越攻击。

### 2. 🔍 全局高亮安全搜索 (Global Search Engine)
构建了支持跨模块聚合搜索的检索引擎：
* **跨表联合检索**：支持对 `Project` (项目)、`Record` (实验记录)、`Attachment` (附件) 进行全局关键字模糊检索。
* **数据越权拦截**：深度集成 RBAC 权限体系，所有搜索 SQL 均强制绑定当前用户的 `accessibleProjectIds`，从数据层物理隔绝越权访问。
* **动态高亮与 XSS 防护**：后端动态计算并剪裁最佳上下文 Snippet，配合 `escapeHtml` 实体转义机制，在保障前端高亮渲染体验的同时，从源头封杀跨站脚本注入。

### 3. 📊 多格式报表引擎 (Export Engine)
为科研记录提供灵活的数据留存能力：
* **多维格式支持**：
  * 基于 `OpenPDF` 实现 PDF 导出，支持动态加载中文字体彻底解决乱码问题。
  * 基于 `Apache POI` 实现 Excel 数据导出，支持自适应列宽与标准表格样式渲染。
  * 支持轻量级 UTF-8 Markdown 导出。
* **流式二进制响应**：前端封装 `Blob` 流式接收与自动触发下载机制，后端采用直接写入 `HttpServletResponse` 输出流的方式，避免巨型报表导出时导致的 OOM 内存溢出。

### 4. 🛡️ 工业级质量保障 (QA & Testing)
摒弃了传统的“点点点”人工测试，全面引入自动化质量体系：
* **万级数据性能压测 (10k Performance Test)**：
  * 基于 `JdbcTemplate` 构建自动化批量灌库脚本。
  * **实测数据**：在 H2 内存数据库下，针对 10,000 条主表记录与 2,000 条关联附件执行跨表 `LIKE` 复杂查询，响应耗时稳定在 **966 ms** 以内，远超预期 3 秒阈值。
  * **隔离设计**：采用特殊的 `perf-` 前缀 ID 隔离测试数据，压测后精确清理，实现测试环境零污染。
* **Playwright E2E 自动化主链路测试**：
  * 构建了 `p5-core-flow.spec.js` 自动化测试流。
  * 实现了：模拟登录 ➡️ 进入工作台 ➡️ 无头浏览器模拟文件上传 ➡️ 触发全局搜索 ➡️ 验证检索结果 ➡️ 拦截导出 PDF 下载事件的 **全主链 100% 自动跑通**。
