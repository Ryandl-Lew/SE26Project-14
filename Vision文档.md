## 2. 项目愿景

### 2.1 问题陈述

生物实验研究人员在日常工作中面临以下痛点：

- **实验记录分散**：实验计划、原始数据、分析结果分散在纸质笔记本、Excel表格、个人电脑等多个位置，难以统一管理和回溯。
- **数据采集遗漏**：实验数据往往需要在特定时间点采集（如每隔2小时取样检测），研究人员容易遗忘或错过关键采集节点，导致实验失败或数据不完整。
- **数据分析门槛高**：实验结束后需要对数据进行统计分析、相关性分析、数学模型拟合等，部分研究人员缺乏编程或统计背景，分析效率低下。
- **协作困难**：团队实验中，成员之间数据共享不便，记录变更无迹可寻，难以追溯问题。
- **知识孤岛**：实验流程和数据仅在小范围内流通，难以参考和借鉴他人的实验设计。

### 2.2 愿景陈述

**为生物实验研究人员打造一站式实验管理平台**，覆盖"计划→提醒→记录→分析→分享"全流程，通过智能化提醒确保数据采集不遗漏，借助AI Agent降低数据分析门槛，支持多人协作与公开数据共享，让研究人员专注于实验本身而非数据管理。

### 2.3 核心价值主张

> 让每一次实验都有迹可循，让每一份数据都发挥价值。

## 3. 目标用户

| 用户角色 | 描述 | 核心诉求 |
|----------|------|----------|
| 实验负责人 | 设计实验方案、统筹实验进度的研究员/导师 | 规划实验流程、分配任务、查看整体进度和结果 |
| 实验操作员 | 具体执行实验操作、采集数据的学生/实验员 | 接收提醒、快速记录数据、查看历史记录 |
| 数据分析员 | 对实验数据进行统计分析的研究人员 | 导入/导出数据、选择模型分析、生成报告 |
| 协作者 | 参与同一实验项目的其他人员 | 查看实验日志、协同填写数据、了解变更历史 |
|实验设计者|构思试验方案或尝试复现实验内容的研究员| 查询相关实验，跟踪实验步骤|

## 4. 系统范围

### 4.1 本次迭代范围（核心功能）

| 功能模块 | 描述 | 优先级 |
|----------|------|--------|
| 项目管理 | 创建、编辑、删除实验项目，定义项目基本信息 | P0 |
| 数据记录 | 以文字条目记录实验数据，支持附图 | P0 |
| 数据采集提醒 | 预设采集时间，到期推送提醒通知 | P0 |
| Agent数据分析 | 选择数学模型，调用Agent进行优化分析和相关性分析，生成实验报告 | P1 |
| 数据上传与查询 | 上传实验数据，联网查询公开实验流程与数据 | P1 |

### 4.2 进阶需求（后续迭代）

| 功能模块 | 描述 | 优先级 |
|----------|------|--------|
| 图片识别自动填表 | Agent识别实验图片内容，自动填充数据条目 | P2 |
| 多人协作 | 多人共同维护同一实验项目，记录数据填写与改动日志 | P2 |
| 美化与创意功能 | 待定 | P3 |

### 4.3 非功能性需求

| 类别 | 要求 |
|------|------|
| 客户端形态 | Web浏览器优先（响应式），后续可扩展为PWA/App |
| 兼容性 | 适配PC端、平板、手机等不同分辨率 |
| 性能 | 支持≥10,000条记录，≥100并发用户，核心操作平均响应时间≤3秒（Agent分析除外） |
| 容错 | 编辑状态下崩溃时，本地自动恢复未保存草稿 |

## 5. 成功标准

| 编号 | 标准 | 衡量方式 |
|------|------|----------|
| SC1 | 用户可在3分钟内完成一个实验项目的创建与数据条目定义 | 用户测试计时 |
| SC2 | 提醒通知在预设时间±1分钟内送达 | 系统测试 |
| SC3 | Agent分析功能可对≥5种常见数学模型进行拟合与优化 | 功能验证 |
| SC4 | 在100并发用户下，数据记录操作响应时间≤3秒 | 压力测试 |
| SC5 | 编辑器崩溃恢复成功率≥95% | 异常测试 |

## 6. 假设与约束

### 6.1 假设
- 用户具备基本的实验方法和数据记录知识
- 用户设备具备网络连接能力
- Agent服务（大语言模型API）可稳定调用

### 6.2 约束
- 开发周期为单次迭代
- 团队规模为小组开发
- Agent调用受限于第三方API的可用性和成本
- 公开数据查询受限于可访问的开放数据库

## 7. 风险

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| Agent API不稳定或成本过高 | 数据分析功能无法使用 | 预留本地统计分析兜底方案 |
| 提醒推送在移动端受限 | 用户错过采集时间 | 提供短信/邮件等多通道提醒 |
| 多人协作数据冲突 | 数据不一致 | 采用乐观锁+变更日志机制 |
| 大量图片存储压力 | 存储成本高 | 图片压缩+云存储+定期清理策略 |
|数据类型上传管理|数据类多样难以统一|用户自定义类|
